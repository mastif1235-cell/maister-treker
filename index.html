Посилання на візитку LNET (сторінка з контактами диспетчера) — вставте сюди URL вашого вже опублікованого сайту-візитки. За цим посиланням формуватиметься QR-код у вкладці «Заявки».
Посилання на сторінку договору. Це окремий файл (dogovor-view.html) — розмістіть його на GitHub Pages чи Netlify, і вставте сюди адресу. Тоді QR у договорі відкриватиме сторінку з даними абонента замість того, щоб кодувати текст напряму — так QR розпізнається набагато надійніше в будь-якому сканері. Якщо залишити порожнім, QR все одно працюватиме по-старому (текстом).
Синхронізація — Заявки ▾
Ключ передається в кожному запиті. Щоб він реально захищав таблицю — додайте перевірку цього ж значення у своєму Apps Script (інакше поле просто ігнорується сервером).
Як налаштувати Google Таблицю
1. Створіть нову Google Таблицю (та/або Google Sheets на Google Drive).
2. Розширення → Apps Script → вставте код з кнопки нижче.
3. Розгорнути → Нове розгортання → тип «Веб-застосунок» → Виконати від «Я», доступ «Будь-хто».
4. Скопіюйте URL і вставте у відповідне поле.
5. Аркуші «Заявки» і «Зміни» створяться автоматично при першому записі.

Для окремого Excel/Google Sheets для змін:
Створіть ще одну таблицю, додайте туди той самий Apps Script і вставте URL нижче.
✈️ Telegram-бот ▾
Необов'язково. Один бот — три різні призначення нижче, кожне вмикається окремо своїм Chat ID. Якщо якесь поле лишити порожнім — відповідна функція просто не працює, решта не постраждає.
1️⃣ Диспетчеру — за кнопкою, вручну
Кнопка "✈️ Диспетчеру" на заявці (у списку і в калькуляторі) одразу надсилає ТЕКСТ цієї заявки диспетчеру в особисті (без фото — воно й так є в бекап-групі нижче) — без відкриття системного меню "Поділитися". Якщо заповнені обидва диспетчери — при натисканні кнопки застосунок запитає, кому конкретно надіслати (одному з двох чи обом одразу).
Якщо впишете сюди й свій особистий Chat ID (той самий, що і в розділі "Місячний звіт" нижче) як одного з "диспетчерів" — копія теж прилітатиме і вам. Спеціальної функції "копія собі" немає — це просто ще один Chat ID у списку.
2️⃣ Резервна копія — тихо, автоматично
При збереженні кожної заявки текст і фото додатково тихо відправляються в закриту групу. Фото й далі зберігається на телефоні як звичайно — це лише додаткова копія на випадок, якщо локальна колись загубиться. Якщо локального фото не буде — застосунок сам підтягне його назад із цієї групи.
Для заявок, які вже є в базі, але ще не потрапили в групу (створені до налаштування бота). Займає час — надсилає по одній заявці з паузою, щоб не впертися в ліміти Telegram. Заявки, які вже позначені ☁️✅, повторно не надсилаються.
Видаляє й наново надсилає геть усі заявки, включно з уже надісланими — щоб у групі точно опинилась актуальна версія кожної (текст + фото + повний JSON-файл). Використовуйте, якщо група збиралась ще до появи JSON-файлу нижче, або якщо просто хочете "почистити й пересинхронізувати" все з нуля.
3️⃣ Місячний звіт — раз на місяць, автоматично
1-го числа кожного місяця бот сам надсилає вам особисто короткий звіт за щойно завершений місяць: скільки змін/годин, скільки заявок і на яку суму, яке обладнання і скільки встановлено.
Той самий Chat ID використовує й кнопка "✈️ Надіслати звіт по змінах у Telegram" на екрані "Зміни" — там можна надіслати звіт саме по змінах за будь-який обраний місяць, коли вам зручно, а не лише 1-го числа.
📖 Детальна інструкція: як усе налаштувати
Крок 1 — створити бота (один раз, обслуговує все три функції):
1. У Telegram знайдіть офіційного бота @BotFather.
2. Надішліть йому команду /newbot, дайте ім'я і юзернейм (має закінчуватись на "bot").
3. BotFather надішле токен виду 123456789:AAExxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx — скопіюйте у поле "Токен бота" вище.

Крок 2 — дізнатись Chat ID (для кожного окремо):
• Диспетчер: нехай диспетчер відкриє чат з вашим ботом і напише йому будь-що (наприклад "привіт"). Потім відкрийте в браузері (з токеном уже вставленим):
https://api.telegram.org/bot<ТОКЕН>/getUpdates
У відповіді знайдіть "from":{"id":987654321 — це Chat ID диспетчера (число без мінуса). Якщо диспетчерів двоє — повторіть те саме з другим диспетчером і впишіть обидва ID через кому в поле вище.

• Група-архів: створіть закриту групу, додайте бота учасником, напишіть у групі будь-яке повідомлення, тим самим способом через getUpdates знайдіть "chat":{"id":-1001234567890 — Chat ID групи (з мінусом, для груп завжди від'ємний).

• Ваш особистий: напишіть боту від свого акаунту, так само гляньте "from":{"id":...} у відповіді — це ваш Chat ID.

Крок 3 — перевірка:
Після заповнення кожної пари полів тисніть відповідну кнопку "🧪 Перевірити" — якщо прийшло тестове повідомлення, все правильно.

Що можна додати самому (для довідки):
Уся логіка живе у функціях sendToTelegramChat(), backupTicketToTelegram() і buildMonthlyTelegramReport() у файлі. Наприклад, щоб додати ще одне авто-повідомлення — можна викликати sendToTelegramChat(chatId, текст) у будь-якому потрібному місці коду (наприклад, після видалення заявки, або коли тег "аварія" додається до заявки).
Синхронізація — Зміни (окремий файл) ▾
Необов'язково. Якщо залишити порожнім — зміни будуть синхронізуватися в ту саму таблицю, що й заявки.

ℹ️ "Відправити всі зміни" надсилає кожну зміну окремим запитом (дублікати за ID скрипт пропустить сам) — масової синхронізації за один запит цей скрипт не підтримує, але працює надійно.

📅 Щоденні бекапи ▾
При першому відкритті застосунку за день автоматично скачується справжній файл (master-tracker-backup-РРРР-ММ-ДД.json) у папку «Завантаження» на телефоні — саме він переживе очищення кешу/даних сайту чи перевстановлення. Такі файли в «Завантаженнях» самі не видаляються — це реальні файли, час від часу почисти старі вручну. Список нижче — це окрема швидка копія за останні 10 днів усередині самого застосунку (для зручного перегляду й відновлення в один дотик), вона зникне разом з рештою, якщо почистити дані сайту.
Якщо браузер запитає дозвіл на автозавантаження файлів для цього сайту — дозволь, інакше щоденний файл не збережеться.
